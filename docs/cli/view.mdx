# opm view / opm whois

Show author profile, OPM ENS records, and published packages. Resolves ENS identity, reads OPM text records, and displays on-chain reputation.

## Usage

<CodeGroup>

```bash View by ENS name
opm view vitalik.eth
```

```bash Whois (auto-appends .eth)
opm whois vitalik
```

</CodeGroup>

<Note>
<code>opm view &lt;name.eth&gt;</code> shows author profile. <code>opm view &lt;pkg&gt;</code> (without <code>.eth</code>) delegates to <code>opm info</code>.
</Note>

## Output

### Identity

- **ENS name** — Resolved identity
- **Address** — Wallet address
- **Bio** — ENS description text record
- **URL** — ENS url record
- **GitHub** — <code>com.github</code> text record
- **Twitter** — <code>com.twitter</code> text record
- **Email** — ENS email record
- **Avatar** — Rendered from ENS avatar record

### Author Stats

- **Packages published** — Count from OPMRegistry
- **Avg reputation** — Risk badge (lower = better)

### OPM ENS Records

If the author has published via OPM, the following text records are displayed:

- **opm.version** — Latest version published
- **opm.checksum** — Tarball checksum
- **opm.fileverse** — Fileverse content hash / report URI (clickable link)
- **opm.risk_score** — Aggregate risk score
- **opm.signature** — ECDSA signature
- **opm.contract** — OPM Registry contract address (clickable BaseScan link)
- **opm.packages** — Comma-separated list of all packages published by this author

### Published Packages

For each package: name, version, risk score, checksum, signature status, and report URI link.

<Note>
No environment variables are required. ENS resolution and text record reading use public resolvers.
</Note>
