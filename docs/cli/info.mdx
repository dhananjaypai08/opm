# opm info

Show on-chain security information and ENS text records for a package. Displays author, checksum, signature, ENS name, report URI, agent scores, aggregate score, safest version, and OPM ENS records.

## Usage

<CodeGroup>

```bash Latest version
opm info lodash
```

```bash Specific version
opm info lodash@4.17.21
```

</CodeGroup>

## Output

### On-Chain Data

| Field | Description |
|-------|-------------|
| Author | Wallet address (with ENS resolution when available) |
| Checksum | SHA-256 hash of the tarball |
| Signature | ECDSA signature (verified or unverified) |
| ENS name | Author identity (e.g. <code>vitalik.eth</code>) |
| Report URI | Link to Fileverse audit report |
| Agent scores | Per-agent risk scores and reasoning |
| Aggregate score | Mean risk score (0-100) |
| Safest version | Lowest-risk version in recent releases |

### ENS Records

If the package author has an ENS name, OPM reads and displays `opm.*` text records:

| Record | Description |
|--------|-------------|
| `opm.version` | Latest version from ENS |
| `opm.checksum` | Checksum from ENS text record |
| `opm.fileverse` | Fileverse content hash (clickable link to audit report) |
| `opm.risk_score` | Risk score from ENS |
| `opm.signature` | Signature from ENS |
| `opm.contract` | OPM Registry contract address |
| `opm.packages` | All packages published by this author |

Per-package records (`opm.pkg.<name>.*`) are read first, falling back to author-level records.

## Links

- **Fileverse report** — Full AI audit from on-chain report URI or ENS `opm.fileverse` record
- **BaseScan** — Contract and transaction links

<Note>
If the package is not in the OPM registry, a warning is shown. No env vars are required.
</Note>
